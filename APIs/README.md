يعني : ✔️ يعني:

إيه الطلبات المتاحة (endpoints)

مجموعة عناصر بتخلي التطبيق والسيرفر يتواصلوا مع بعض. الAPI
1️⃣ البروتوكول (Protocol)
ده الطريقة اللي البيانات هتتنقل بيها
مثال: HTTP / HTTPS / WebSocket
مسؤول عن:
شكل الرسائل
طريقة الإرسال والاستقبال
الأمان والاتصال
2️⃣ الـ Endpoints
ده المسارات اللي بيتم الوصول لها على السيرفر
كل endpoint بيعمل وظيفة معينة
مثال:
/api/users → تجيب بيانات المستخدمين
/api/login → تسجيل دخول
3️⃣ Requests (الطلبات)
اللي التطبيق بيبعته للسيرفر
ممكن تكون: GET, POST, PUT, DELETE
بيحتوي على:
Headers (زي Authorization)
Body (البيانات المرسلة لو POST مثلا)
s هو فعليًا اللي بنسميه HTTP Methods أو طرق الطلب في HTTP.

🔹 أشهر HTTP Methods:
Method	الاستخدام
GET	تجيب بيانات من السيرفر (Read)
POST	تبعت بيانات للسيرفر عشان ينشئ حاجة جديدة (Create)
PUT	تحدّث بيانات موجودة بالكامل (Update كامل)
PATCH	تحدّث جزء من البيانات (Update جزئي)
DELETE	تمسح بيانات من السيرفر (Delete)
🔹 العلاقة بالـ API:
كل endpoint في الـ API بيقبل واحد أو أكتر من الـ Methods دي
طريقة التطبيق يبعت البيانات أو يستقبلها تتحدد بالـ HTTP Method
4️⃣ Responses (الردود)
اللي السيرفر بيرجعه للتطبيق
غالبًا بيكون JSON
بيحتوي على:
Data (المطلوب)
Status (نجاح أو خطأ)
Error message لو حصل مشكلة
5️⃣ Documentation (توثيق)
دليل الـ API للمطورين
بيشرح:
كل endpoint موجود إيه
نوع الطلب المطلوب
شكل الـ response
مثال: Swagger / Postman docs

🔹 ببساطة جدًا:

👉 الاتنين Protocols (بروتوكولات)

👉 هو مجموعة قواعد بتنظم إزاي الأجهزة تتكلم مع بعض

زي:

🔥 نطبق ده على كل واحد:
🟡 HTTP

👉 Protocol بيحدد:

إزاي تبعتي request
شكل الـ request عامل إزاي

📌 يعني:

🔵 WebSocket

👉 Protocol بيحدد:

إزاي نفتح connection مستمر
إزاي نبعت ونستقبل data في أي وقت

📌 يعني:
قواعد اتصال مفتوح طول الوقت

💡 تشبيه بسيط:
HTTP = تبعتي إيميل وتستني الرد 📧
WebSocket = مكالمة تليفون مفتوحة 📞
🧠 الخلاصة:

👉 الاتنين مش كود ولا مكتبات
👉 الاتنين قوانين/قواعد للتواصل بين client و server

تمام، دي أهم نقطة فعلًا 👌 خلّيني أوضحلك **المشترك والاختلاف بين HTTP و WebSocket بشكل بسيط وواضح:

🔹 أولًا: الحاجات المشتركة بينهم
الاتنين بيشتغلوا فوق TCP connection
الاتنين بيستخدموا في التواصل بين Client (زي الموبايل/المتصفح) و Server
الاتنين ممكن تبعتي وتستقبلي بيانات من خلالهم
🔥 الاختلاف الأساسي (أهم نقطة)

هو ببساطة:
👉 اتصال ثابت وموثوق بين جهازين على الإنترنت
(زي الموبايل بتاعك والسيرفر)

🧠 الفكرة ببساطة:

قبل ما البيانات تتبعت، لازم يحصل:

اتفاق بين الطرفين (فتح اتصال)
بعد كده يبدأوا يتبادلوا البيانات
وفي الآخر يقفلوا الاتصال

TCP Connection =
👉 قناة اتصال مضمونة بين جهازين
👉 بتتأكد إن الداتا توصل صح وكاملة


🔹 TCP اختصار لإيه؟

Transmission Control Protocol

👉 يعني:
Transmission = نقل
Control = تحكم
Protocol = قواعد

📌 المعنى كله:
👉 بروتوكول مسؤول عن نقل البيانات بشكل منظم ومضمون

🔹 أنواع الـ Connections التانية

في عالم الشبكات، أشهر نوعين:

🟡 1. TCP
Connection-based (لازم يفتح اتصال الأول)
موثوق (Reliable)
بيضمن وصول البيانات كاملة وبالترتيب

📌 بيستخدم في:

HTTP
WebSocket
تحميل الملفات / APIs
🔵 2. UDP
Connectionless (مفيش اتصال ثابت)
أسرع لكن مش مضمون
ممكن data تضيع أو توصل غلط

📌 بيستخدم في:

الألعاب أونلاين 🎮
الفيديو لايف 📺
VoIP (المكالمات)

TCP بيعمل حاجة اسمها 3-way handshake:

Client → SYN (عايز أفتح اتصال)
Server → SYN-ACK (موافق)
Client → ACK (تمام نبدأ)

👉 بعد كده يبدأوا يبعتوا الداتا

🧠 الخلاصة:
TCP = آمن ومضمون (زي رسالة بعلم الوصول 📩)
UDP = سريع بس مش مضمون (زي رسايل واتساب لو النت وحش 😅)
